JavaScript interview question and answers

1. Difference Between 'var' 'let' and 'Const'

var , let & const all are use for declaring variable.but there are so many diffrence between them. We are showing the key differences between them

var act as a global variable. if we declare a variable using var , it can be accessable by all the function . which can create data loss and complexity.
let is the latest Es6 feature of declaring variable. if we declare a variable using let.It will work only inside the area of function. We can declare a variable only once by let. thats the good practice.
const is the feature of ES6 .if we declare a varibale using const then,the value cant be modified or redeclared.It act like a constant value.It also work inside its scope. The Variable declared with const can not be modiified.using const is a good practice.

2. map() , foreach() , filter() , find()

map , foreach , filter , find are the arrowfunction method for an array. these function can be used only on the variable which are array

Map(num => num*2) Map function takes each of the array element and transform the array by following equation(n*2) and return the modified array as an array.The modification doesnot affect the orginal array.
forEach(num => console.log(num)) foreach function works same as map. but the foreach function doesnt return any value, and also it doesnot affect the orginal array also.
filter(num => num > 30) filter function contain a condition . it takes all element of the array and check every element with the conndition. It make another array with all the elements that checked as true. the array is returned by filter function.
find(num => num > 30) find works same as filter method. but it returns the first true value of the array element which makes the condition true.find returns only a number which is the true value for the condition.

3. Why we use Template String??

template string is one of the best feature of ES-6. The Template String can print the newline without declaring by \n , and it can takes variable and any expression inside it.

Reason to use template String 1. Template String can detect newline .
2. We can use any variable inside it by using ${varName}this.
3. We can use any type of expression and code inside a string.
4. We dont need to concat strings in a complex way. The Dynamic String making is now quite easy for the template string feature.
